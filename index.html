Hospital Information Management System - Website Overview & User Guide

Overview Features User Roles Subdomains Login & Dashboard Flow

1. Overview

Hospital360 HIMS is a multi-tenant hospital management system designed to manage patient records, staff, appointments, billing, pharmacy, labs, and other hospital operations efficiently. The system supports multiple hospitals under a single SaaS platform while maintaining secure access for each hospital and role.

2. Key Features

  • Patient Management (registration, records, history)
  • Doctor & Staff Management
  • Appointment Scheduling
  • Billing & Invoicing
  • Pharmacy & Inventory Management
  • Laboratory Management
  • Role-based Dashboards and Permissions
  • Multi-Hospital Support (tenant system)
  • Super Admin Dashboard for SaaS owner
  • Audit Logs & Security Notifications

3. User Roles

The system defines different roles with specific permissions:

  • Super Admin: Full SaaS control, manage all hospitals, subscriptions, users, roles.
  • Hospital Admin: Manage staff, departments, billing, and hospital-level configurations.
  • Consultant Doctor: Access patient consultation, prescriptions, and reports.
  • Junior Doctor: Assist in consultations and patient follow-ups.
  • Receptionist: Handle patient registration and appointments.
  • Pharmacy Staff: Manage medicine inventory and prescriptions.
  • Lab Technician: Handle lab tests and reports.

4. Subdomains

The system uses subdomains to separate SaaS admin and hospital portals:

  • hospital360.tech - Main public website and login page
  • app.hospital360.tech - Tenant portal for hospital staff (all roles)
  • super.hospital360.tech - SaaS Super Admin dashboard
  • {hospital_code}.hospital360.tech (optional) - Individual hospital tenant portal

5. Login & Dashboard Flow

  1. All users log in at https://hospital360.tech/accounts/login/.
  2. System authenticates the user and checks their role.
  3. Super Admin → redirected to https://app.hospital360.tech/dashboard/.
  4. Hospital Staff → redirected to hospital-specific portal: {hospital_code}.hospital360.tech.
  5. Within the hospital portal, dashboards and features are role-based.
  6. Login audit emails and notifications are sent upon successful login.

This flow ensures a single login page while maintaining secure access per hospital and role.

6. Additional Notes

  • All hospital staff and admins use the same portal; roles determine their dashboard access.
  • No role-based subdomains exist; roles are managed internally in Django.
  • The system is fully multi-tenant and scalable for new hospitals.
  • Audit logs track user login, role access, and actions for security and compliance.
